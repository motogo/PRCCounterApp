set_base_seconds(self, seconds)

source code 

This function sets a base in whole seconds for the following time tags (future time tags value will be relative to this value). The base seconds is used to eliminate the problem of lost resolution over long time intervals (1000 seconds and more). Because the time tags are represented as double precision floating point numbers when the time value gets above 1000 seconds the least significant bit becomes 1 ps and from there on resolution will be lost. Setting the base seconds of the time tags to 1000 (for example) with a call to set_base_seconds(1000) will reduce the time tag value to a value close to 0 with least significant bit of 1 fs. Default value of base seconds is 0.

Parameters:
  • seconds (unsigned int) - Base seconds for the following time tags. The value must be less than or equal to the seconds' part of the last time tag
Returns: bool
Returns False if setting failed, True if setting was successful.

set_input_prescale(self, signal, prescale)

source code 

This function sets the hardware input prescaling of a measured signal to divide the input by a power of 2 value between 1 and 1024. The prescaling must be high enough to reduce the measured signal to a frequency under 4MHz. NOTE: Clock and Arm signals do not support prescaling (same as GT_DIV_1).

Parameters:
  • signal (string) - see Signal
    • GT_SIG_A - Input channel A
    • GT_SIG_B - Input channel B
    • GT_SIG_CLK - Clock input
    • GT_SIG_ARM - Arm Input
  • prescale (int) - see prescale
    • GT_DIV_AUTO - Select automatically the minimal prescaling value needed to bring the signal under 4MHz
    • GT_DIV_1 - divide by 1 (no prescaling)
    • GT_DIV_2 - divide by 2
    • GT_DIV_4 - divide by 4
    • GT_DIV_8 - divide by 8
    • GT_DIV_16 - divide by 16
    • GT_DIV_32 - divide by 32
    • GT_DIV_64 - divide by 64
    • GT_DIV_128 - divide by 128
    • GT_DIV_256 - divide by 256
    • GT_DIV_512 - divide by 512
    • GT_DIV_1024 - divide by 1024
Returns: bool
Returns False if setting failed, True if setting was successful.

set_T0_mode(self, arm, rel)

source code 

This function configure the T0 mode. The arm value selects if the reference time (t0) will be generated from the block arming tag (with 20ns resolution) or from the first time tag (full channel resolution). The rel value selects if the T0 value will be subtracted from all time tags.

Parameters:
  • arm (bool) -
    • True - t0 generated from block arm
    • False - t0 generated from first time tag
  • rel (bool) -
    • True - t0 will be subtracted from all time tags
    • False - t0 will not be subtracted from all time tags
Returns: bool
Returns False if setting failed, True if setting was successful.

real_time_output(self, out, seconds, frac)

source code 

This function sends to the board a real time value at which to generate an output pulse. The fraction will be converted to the nearest 10 nsec interval. Notes:

  • 1. Only future times are valid.
  • 2. Calls should be in advancing time order for each output.
  • 3. If two time values are in the same 10 nsec interval the second one is ignored.
  • 4. Up to 64 time values per output can be pending at any time.
  • 5. The function will fail if the output source is not set to GT_OUT_REALTIME (see function set_output).
Parameters:
  • out - Output number to select (0 or 1).
  • seconds - Real time seconds value.
  • frac - Real time fraction (0 to less than 1).
Returns: bool
Returns True if channel is enabled, False otherwise.
